DMM WEBCAMPコンテンツ【システム設計】 DMM WEBCAMPの学習コンテンツ[システム設計]の研修課題。

<h1>ながのCAKE</h1>

<h2>◆概要</h2>
  長野県にある小さな洋菓子店「ながのCAKE」の商品を通販するためのECサイト開発。<br>

<h2>◆案件の背景</h2>
  元々近隣住民が顧客だったが、昨年始めたInstagramから人気となり、全国から注文が来るようになった。<br>
  InstagramのDMやメールで通販の注文を受けていたが、情報管理が煩雑になってきたため、管理機能を含んだ通販サイトを開設しようと思い至った。<br>

<h2>◆通販について</h2>
・通販では注文に応じて製作する受注生産型としている。<br>
・現在通販での注文量は十分に対応可能な量のため、1日の受注量に制限は設けない。<br>
・送料は1配送につき全国一律800円。<br>
・本来は軽減税率により宅配物は税率8%ですが、今回は10%で統一する。<br>
・友人や家族へのプレゼントなど、注文者の住所以外にも商品を発送できる。<br>
・支払方法はクレジットカード、銀行振込から選択できる。<br>

<h2>◆実装機能</h2>
★マークは実装必須の要件、☆マークは任意の実装要件。

<h3>[会員側実装機能]</h3>
★顧客は会員登録、ログイン・ログアウト、退会ができる<br>
★会員のログインはメールアドレスとパスワードで行う<br>
☆会員がログインしている状態かどうか、ページのヘッダーにユーザ名を表示する<br>
★サイトの閲覧はログインなしで行える<br>
★商品をカートに入れ、1度に複数種類、複数商品の購入ができる。また、カート内の商品は個数変更・削除ができる<br>
★カートへの商品追加はログインなしでは行えない<br>
★クレジットカード情報はシステム上保持しない<br>
☆会員は配送先を複数登録しておくことが可能である<br>

<h4>会員はマイページから下記が行える</h4>
★ユーザ情報の閲覧・編集<br>
★注文履歴の閲覧<br>
☆配送先の閲覧・編集<br>

<h4>注文履歴一覧には下記の情報が表示されること</h4>
★注文日<br>
★配送先<br>
★支払金額（商品合計＋送料）<br>
☆注文ステータス<br>

<h4>注文履歴詳細には下記の情報が表示されること</h4>
★注文日<br>
★配送先<br>
★支払方法<br>
☆注文ステータス<br>
★商品の注文内容詳細（商品名、単価、個数、小計）<br>
★請求情報（商品合計、送料、支払金額）<br>

<h4>会員登録時、下記の情報をユーザ情報として入力できる</h4>
・名前（姓・名）<br>
・名前（カナ姓・カナ名）<br>
・郵便番号<br>
・住所<br>
・電話番号<br>
・メールアドレス<br>
・パスワード<br>

<h4>その他</h4>
☆商品は税込価格で表示される<br>

<h3>[管理者側(店側)実装機能]</h3>
★管理者用メールアドレスとパスワードでログインできる<br>
（管理者用メールアドレスとパスワードは事前に「/nagano_cake/db/seeds.rb」に登録している）<br>

<h4>商品について、下記が行える</h4>
★新規追加、編集、閲覧<br>
☆販売停止<br>

<h4>商品情報は下記のものを持つ</h4>

★商品説明文<br>
☆ジャンル<br>
★税抜価格<br>
★商品画像<br>
☆販売ステータス<br>
★会員登録されているユーザ情報の閲覧、編集、退会処理が行える<br>

<h4>ユーザの注文履歴が一覧・詳細表示でき、下記の情報が表示される</h4>

★配送先<br>
★支払方法<br>
★料金（商品合計、送料、請求金額合計）<br>
